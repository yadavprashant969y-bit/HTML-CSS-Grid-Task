Answers
1. Why do we check browser support?

We check browser support to ensure our website looks good even on older browsers that might not understand modern CSS features like Grid.

2. What happens if Grid is not supported?

If Grid is not supported and we don't have a fallback, the layout might break. With @supports, we can provide alternative styles (like block or flex) for older browsers.

3. Is this needed for all projects?

It depends. For modern projects targeting mostly new devices, it might not be strictly necessary as Grid support is very high (98%+). But for government or enterprise sites supporting old systems, it is crucial.

4. Who uses this in real projects?

Developers who need to support legacy browsers (like old Internet Explorer versions) use this technique.

5. What does safe CSS mean?

Safe CSS means writing code that doesn't break if a feature isn't supported. It "fails gracefully" using fallbacks.
